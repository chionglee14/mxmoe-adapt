# 项目管理与里程碑

## 角色建议

团队最多3人时建议分工：

- 负责人：技术路线、C500环境、FlagGems上游协调和最终发布。
- Kernel负责人：联合调优、C500 Kernel和性能分析。
- 工程与社区负责人：基准、CI、文档、GitLink镜像和Issue维护。

单人申报时按同样工作流执行，只是不区分人员。

## 里程碑

### M0：可运行申报版本

- 开源仓库、许可证、README、申报书。
- 环境采集、路由特征、搜索空间、派发与漂移检测。
- CPU测试通过。

### M1：C500基线

- 固定MXMACA环境和FlagGems提交。
- FP16/BF16小Shape正确性。
- 不少于10组Shape/路由基线。

### M2：联合调优

- 对齐块和GEMM配置联合搜索。
- 不少于30组实测。
- 第一版已验证配置数据库。

### M3：专用优化路径

- Decode/Prefill至少完成一条专用路径。
- 与通用FlagGems公平对比。
- 完成消融实验：只调GEMM、只调对齐、联合调优。

### M4：集成与发布

- 模型层和单卡端到端验证。
- FlagGems Issue/PR。
- v0.1正式发布、GitLink镜像、项目报告。

## 决策门槛

- 没有C500环境信息：不进入性能阶段。
- 正确性未通过：不排名、不发布加速比。
- 环境指纹改变：先运行守护Shape，再决定是否全量重调。
- 首期时间不足：砍掉FP8、多卡和全模型，不砍正确性与可复现性。
