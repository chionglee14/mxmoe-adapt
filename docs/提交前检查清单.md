# 提交前检查清单

## 必填信息

- [ ] 负责人姓名、单位、联系方式已填写。
- [ ] 团队不超过3人，每人职责明确。
- [ ] GitHub或Gitee公开仓库地址可访问。
- [ ] Apache-2.0许可证在仓库根目录。
- [ ] GitLink镜像计划已写入申报内容。
- [ ] 项目名称、摘要、方向和周期在所有材料中一致。

## 已有基础

- [ ] README中的命令可在干净环境执行。
- [ ] CPU单元测试通过。
- [ ] 示例配置明确标注未实测、`verified=false`。
- [ ] 若已有C500结果，环境JSON和原始结果一并提交。
- [ ] 不把其他GPU结果表述为C500结果。

## 创新与可行性

- [ ] 明确区别于“FlagGems适配沐曦”的已有工作。
- [ ] 创新点包含路由感知、联合优化、置信回退和漂移检测。
- [ ] 首期边界为单卡C500与FP16/BF16。
- [ ] 性能指标写成目标值，不写成已取得结果。
- [ ] 风险、回退路径和负结果公开机制清楚。

## 第一批申报

- [ ] 在2026年7月31日前提交表单。
- [ ] 提交前再次打开官网确认时间和表单状态。
- [ ] 保存提交成功截图或回执。
- [ ] 若来不及获得可靠C500结果，评估转第二批而非编造数据。
