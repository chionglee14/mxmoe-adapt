# C500 实测手册

本手册用于沐曦曦云 C500 真机。所有命令使用 `python3`，执行顺序遵循“先确认环境，再安装或修改”。

## 1. 只读环境确认

先运行平台提供的状态命令并保存输出：

```bash
mx-smi
python3 --version
python3 -m pip --version
```

进入项目后，先在现有环境中采集信息：

```bash
python3 -m mxmoe_adapt.environment --output results/c500-environment-before.json
```

必须确认：

- 设备确实为曦云 C500；
- Driver、MXMACA、mcPytorch、mcTriton或FlagTree版本可取得；
- `torch.cuda.is_available()`为真；
- `torch.cuda.get_device_name()`与目标设备一致；
- FlagGems当前提交已固定。

未确认前不要升级包，不要覆盖平台镜像。

## 2. 隔离项目环境

优先使用平台已经验证的MXMACA容器或镜像。在不破坏平台依赖的前提下创建独立环境：

```bash
python3 -m venv .venv --system-site-packages
source .venv/bin/activate
python3 -m pip install -e .
python3 -m unittest discover -s tests -v
```

若平台明确要求直接使用镜像中的Python，则不强制创建venv，但必须记录原因和完整包版本。

## 3. 小Shape冒烟

先验证PyTorch Reference自身：

```bash
python3 benchmarks/benchmark_fused_moe.py \
  --candidate mxmoe_adapt.adapters.reference:run \
  --device cuda --dtype fp16 --tokens 2 --experts 4 \
  --hidden-size 64 --intermediate-size 128 --top-k 2 \
  --warmup 2 --repeats 5 \
  --output results/reference-smoke.json
```

再验证FlagGems：

```bash
python3 benchmarks/benchmark_fused_moe.py \
  --candidate mxmoe_adapt.adapters.flaggems:run \
  --device cuda --dtype fp16 --tokens 4 --experts 8 \
  --hidden-size 256 --intermediate-size 512 --top-k 2 \
  --warmup 5 --repeats 20 \
  --output results/flaggems-smoke-fp16.json
```

BF16重复一次。若失败，完整保留错误、环境JSON和提交号，禁止跳过正确性直接测性能。

## 4. 扩大Shape的安全顺序

```text
小合成Shape
-> 中等Shape
-> 单个真实MoE层Shape
-> 多组路由分布
-> 模型层集成
-> 单卡模型端到端
```

每次扩大前估算权重和中间缓冲区显存。若显存不足，减少专家数或使用层级回放，不通过修改结果口径掩盖OOM。

## 5. 路由样本

每个静态Shape至少覆盖：

- 近似均衡路由；
- 高偏斜路由；
- 大量空专家的Decode路由；
- 从真实模型采集并脱敏的Top-K ID统计。

可以公开计数直方图和统计特征，不需要公开用户输入或原始提示词。

## 6. 调优与验证

```bash
python3 -m mxmoe_adapt.search_space --output results/c500-search-space.json
```

搜索空间中的共享存储数值只是过滤代理。每个候选仍须经过：编译成功、正确性通过、重复计时、环境绑定。最优配置写入正式数据库前，将 `verified` 设为 `true` 并附原始结果文件路径。

## 7. 发布检查

- 环境JSON中不存在密码、Token、内网地址或用户数据。
- 结果来自C500，不使用NVIDIA或其他设备代替。
- baseline与candidate代码提交均可追溯。
- 不使用第一次偶然最快值；保留全部重复测量。
- 加速比由同一环境、同一Shape、同一计时口径计算。
- 未达到目标也保留负结果与瓶颈分析。
