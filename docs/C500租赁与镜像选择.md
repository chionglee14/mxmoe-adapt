# C500租赁与镜像选择

## 一、页面选项怎么选

### 硬件

- GPU厂商：沐曦 MetaX
- GPU型号：曦云 C500
- 数量：1张
- 显存：64GB（页面可能显示65536MiB）
- 使用方式：整卡独占，不选择vGPU、切分卡或共享卡
- CPU：最低16核，推荐32核
- 内存：最低64GB，推荐128GB
- 数据盘：最低100GB，推荐200GB
- 租用方式：先按时租4—6小时，环境通过后续租12—24小时

首期算子调优不需要8卡。多卡部署和All-to-All不属于第一阶段。

### 操作系统

- 首选：Ubuntu 22.04 x86_64
- 次选：平台明确验证过MXMACA的麒麟V11 x86_64
- 不选：Ubuntu 18.04、Windows、ARM镜像、纯净裸Ubuntu

### 镜像类型

按以下优先级选择：

1. 官方“MXMACA开发 / mcPyTorch / mcTriton”镜像，且包含完整编译器和开发工具。
2. 官方“AI / vLLM-MetaX”推理镜像。
3. 只有在前两项都没有时，才选MXMACA基础SDK镜像，并要求平台确认可安装mcPyTorch和mcTriton。

不要选择只带驱动、只带`mx-smi`或名称含“CUDA兼容”但没有mcPyTorch/mcTriton的镜像。

## 二、推荐版本

### 优先方案

如果平台有经验证的3.8.x镜像，选择：

```text
MXMACA 3.8.x
mcPyTorch / torch 2.8
mcTriton 3.x（或3.8软件栈配套版本）
Python 3.10或3.12
Ubuntu 22.04 x86_64
```

前提是平台明确说明该镜像与当前C500宿主机Driver兼容。

### 已公开可参考的备用方案

若没有3.8.x开发镜像，可选近期公开使用的：

```text
cr.metax-tech.com/public-ai-release/maca/vllm-metax:
0.20.0-maca.ai3.7.0.107-torch2.8-py310-ubuntu22.04-amd64
```

镜像Tag不要手抄截断，使用平台页面提供的完整版本。宿主机Driver必须由租赁方确认兼容。

项目首期只测FP16/BF16，不选择FP8模型或FP8专用环境。

## 三、可以直接复制给平台客服的需求

```text
用途：沐曦C500上的MXMACA/mcTriton/FlagGems MoE算子开发、编译和性能调优，不是普通模型推理体验。

硬件要求：
1. MetaX曦云C500 64GB整卡独占1张，不使用vGPU或共享切分；
2. x86_64，CPU不少于16核，内存不少于64GB；
3. 数据盘不少于100GB；
4. 支持SSH登录和项目文件上传。

软件要求：
1. Ubuntu 22.04 x86_64；
2. 官方MXMACA开发镜像或官方vLLM-MetaX镜像；
3. 包含mcPyTorch、mcTriton及MXMACA编译工具链；
4. 优先MXMACA 3.8.x配套镜像；若无，则使用经验证的maca.ai3.7.x + torch2.8镜像；
5. 容器MXMACA版本必须与宿主机Driver兼容；
6. 容器内能运行mx-smi、python3，并能编译执行Triton Kernel；
7. 允许创建Python venv或在用户目录执行pip install -e本地项目；
8. 最好能访问GitHub/Gitee；若不能，至少支持SCP/SFTP上传。

请在开通前提供：完整镜像Tag、宿主机Driver版本、MACA版本、Python版本、mcPyTorch版本，以及是否确认mcTriton Kernel可编译运行。
```

## 四、租赁页面常见字段填写

| 字段 | 填写内容 |
|---|---|
| 实例名称 | `mxmoe-c500-dev` |
| GPU | `MetaX C500 64GB` |
| GPU数量 | `1` |
| GPU模式 | `整卡独占` |
| 系统 | `Ubuntu 22.04 x86_64` |
| 镜像分类 | `AI / vLLM-MetaX`或`MXMACA开发` |
| 镜像版本 | `3.8.x配套最新版`；没有则选`0.20.0-maca.ai3.7.0.107-torch2.8-py310-ubuntu22.04-amd64` |
| CPU | `32核`，预算紧张可降至16核 |
| 内存 | `128GB`，预算紧张可降至64GB |
| 数据盘 | `200GB`，最低100GB |
| 网络 | `SSH + 文件上传`，最好允许GitHub/Gitee |
| 租用时长 | `先4—6小时` |
| 备注 | 粘贴上一节客服需求 |

## 五、开通后只需要发给Codex的内容

不要发送密码。建议在本机`~/.ssh/config`配置：

```sshconfig
Host metax-c500
    HostName 平台给出的地址
    Port 平台给出的端口
    User root
    IdentityFile 你的私钥路径
```

然后只告诉Codex：

```text
SSH别名：metax-c500
实例到期时间：YYYY-MM-DD HH:MM
```

如果平台只提供密码登录，在本机终端先完成登录或配置密钥，不要把密码写入项目或聊天记录。

## 六、开通后的第一轮检查

第一轮不安装软件，只执行：

```bash
mx-smi
python3 --version
python3 -m pip --version
python3 -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
python3 -c "import triton; print(triton.__version__)"
```

确认环境真实可用后，再上传项目并运行Triton冒烟测试。
